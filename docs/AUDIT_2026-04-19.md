## Phase 1 — Static analysis (COMPLETE)

**Result: 0 findings after false-positive filtering.**

Scanner at [scripts/audit_scan.py](../scripts/audit_scan.py). Detects four bug classes, all of which we lived through and fixed during the RBC work in sessions 2026-04-18/19:

1. **ROWID-to-static-cell in quarterly formula** — `{ROWID:X}` in a Col="C" quarterly formula where X is a single-cell static input. Resolves to empty at Q2+. (Fixed in commit `68c3569`.)
2. **REF-to-non-existent-RowID** — `{REF:Tab!RowID}` where the target is undefined. Produces `#REF!`. (Fixed in commits `0112fce`, `2f44fad`.)
3. **Duplicate-row collision** — Two RowIDs at the same `(row, col)` cell. RowID cache only indexes one. (Fixed in commit `68c3569`.)
4. **Signed-convention mirror without ABS** — Mirror of a known-signed-negative source (e.g., `UWEX_CRSV` which is stored negative per PD-05) without an ABS wrapper or leading-minus sign flip. (Fixed in commit `2f44fad`.)

Scanner checked 3,142 config rows across 30 tabs. First run surfaced 168 findings, of which 167 were false positives from two sources now filtered:
- `Quarterly Summary`, `Detail`, `Run Metadata`, `Loss Triangles`, `Count Triangles` have RowIDs populated at runtime by VBA (not defined in `formula_tab_config.csv`). Scanner now whitelists these tabs and checks naming conventions instead of CSV presence.
- Leading `-` sign flip (e.g., `=-{REF:...}` on `BS_RI_RECV`) is a legitimate alternative to `ABS()`. Scanner now recognizes both.

**Recommendation — wire into CI.** Add `python3 scripts/audit_scan.py` as a pre-commit check and a pre-packaging gate. Any non-zero output = don't merge / don't ship. This is the Phase 4 "regression assertions" decision at the static level.
